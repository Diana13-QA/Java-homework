Виконання цього завдання потребує наступних кроків. Нижче наведено докладний план дій:

1. Ознайомтесь з API документацією

Перш за все, перегляньте надану API-документацію, щоб зрозуміти, які існують запити (наприклад, GET, POST, PUT, DELETE), їхні параметри та структуру відповіді.

2. Створіть JMeter тест-план

Налаштування JMeter:

  1. Встановіть JMeter, якщо він ще не встановлений.
  2. Додайте тест-план:
    • Створіть новий Test Plan у JMeter.
    • Додайте HTTP Request Defaults для встановлення базового URL (винесіть його у змінну, наприклад, ${BASE_URL}).
    • Використовуйте Config Elements → User Defined Variables, щоб оголосити змінну ${BASE_URL}.

Створення запитів:

  1. Додайте HTTP Request для кожного типу запиту, описаного в документації.
  2. Для імені використовуйте JMeter функцію __RandomString, наприклад:
    scss
    ${__RandomString(8,abcdefghijklmnopqrstuvwxyz,)}

Додайте графіки:

  1. Увімкніть Listener для Aggregate Report.
  2. Додайте Listener для Graph Results.

3. Налаштуйте сценарії навантаження

Налаштування Thread Group:

  1. Створіть першу Thread Group з наступними параметрами:
    • Number of Threads (Users): 100
    • Ramp-Up Period (seconds): 1
    • Loop Count: 1
  2. Повторіть налаштування для:
    • 1000 користувачів, 1 секунди Ramp-Up, 2 повторення.
    • 5000 користувачів, 1 секунди Ramp-Up, 4 повторення.

Експорт результатів:

  • Після кожного запуску збережіть результати у форматі HTML:
    • Увімкніть Generate Dashboard Report в JMeter, вкажіть папку для збереження звіту.

4. Додайте Duration Assertion

  1. Додайте Assertions → Duration Assertion до всіх запитів, щоб встановити максимальний час відповіді (наприклад, 1000 мс).
  2. Налаштуйте нові тести з такими параметрами:
    • 1000 користувачів, Ramp-Up 2 секунди, Loop Count: Infinity, Duration: 30 секунд.
    • 5000 користувачів, Ramp-Up 3 секунди, Loop Count: Infinity, Duration: 60 секунд.
    • 10000 користувачів, Ramp-Up 5 секунд, Loop Count: Infinity, Duration: 180 секунд.

Експорт результатів:

  • Збережіть результати у форматі HTML.

5. Відстеження навантаження ОС

  • Запустіть Task Manager (Windows) або System Monitor (Linux/Mac), щоб відслідковувати CPU, RAM та мережеве навантаження під час виконання тестів.

6. Додайте репорти до Git

  1. Створіть новий репозиторій у GitHub.
  2. Додайте звіти до репозиторію:
    • Створіть структуру папок: /reports/scenario1, /reports/scenario2, тощо.
    • Додайте HTML-звіти та супутні файли.
  3. Закомітьте та пушніть зміни:
    bash
    git init
git add .
git commit -m "Added performance test reports"
git branch -M main
git remote add origin <repository_url>
git push -u origin main

Для створення HTML-звітів з JMeter потрібні такі інструменти:

  1. JMeter – для виконання тестів і генерації .jtl файлів результатів.
  2. Команда в терміналі – для запуску JMeter у режимі CLI та створення HTML-звітів.

Ось як можна це зробити:

1. Підготовка до запуску JMeter:

  • Завантажте Apache JMeter.
  • Створіть тест-план згідно з описом завдання.
  • Збережіть тест-план у .jmx файлі, наприклад, performance_test.jmx.

2. Виконання тестів:

Запустіть JMeter у режимі командного рядка для кожного сценарію:

bash
# Сценарій 1: 100 користувачів, 1 Ramp-Up, 1 Loop Count
jmeter -n -t performance_test.jmx -l results_100.jtl -e -o report_100

# Сценарій 2: 1000 користувачів, 1 Ramp-Up, 2 Loop Count
jmeter -n -t performance_test.jmx -l results_1000.jtl -e -o report_1000

# Сценарій 3: 5000 користувачів, 1 Ramp-Up, 4 Loop Count
jmeter -n -t performance_test.jmx -l results_5000.jtl -e -o report_5000

# Сценарій 4: 1000 користувачів, 2 Ramp-Up, Duration 30 сек
jmeter -n -t performance_test.jmx -l results_duration_1000.jtl -e -o report_duration_1000

# Сценарій 5: 5000 користувачів, 3 Ramp-Up, Duration 60 сек
jmeter -n -t performance_test.jmx -l results_duration_5000.jtl -e -o report_duration_5000

# Сценарій 6: 10000 користувачів, 5 Ramp-Up, Duration 180 сек
jmeter -n -t performance_test.jmx -l results_duration_10000.jtl -e -o report_duration_10000

3. Результат:

Після кожного запуску JMeter створить HTML-звіти в зазначених папках (report_100, report_1000, тощо).

4. Завантаження звітів у GitHub:

  1. Збережіть усі звіти в окремі папки:
    bash
    /reports/
    scenario1_100_users/
    scenario2_1000_users/
    scenario3_5000_users/
    scenario4_1000_users_duration/
    scenario5_5000_users_duration/
    scenario6_10000_users_duration/
  2. Додайте їх у Git:
    bash
    git init
git add reports/
git commit -m "Added performance reports"
git remote add origin <repository_url>
git push -u origin main

Я можу допомогти вам з налаштуванням скриптів для автоматизації цього процесу або згенерувати готові .jmx файли, які ви зможете використати для тестування. На жаль, я не можу створити HTML-звіти без виконання тестів у JMeter, але можу детально пояснити кожен крок.
